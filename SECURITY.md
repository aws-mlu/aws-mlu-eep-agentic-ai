# Security

## Reporting Security Issues

If you discover a potential security issue in this project, please **do not** open a public GitHub issue.

Instead, report it through the [AWS Vulnerability Disclosure Program](https://hackerone.com/aws_vdp) or via [email](mailto:aws-security@amazon.com).

For more details, visit the [AWS Vulnerability Reporting Page](http://aws.amazon.com/security/vulnerability-reporting/).

## Shared Responsibility

This repository contains **educational content** — slide decks, lab instructions, and example notebooks designed for learning. It is not a deployable service or library.

If you use the code examples or lab exercises in this repository as a starting point for your own applications, **you are responsible for the security of what you build**, including:

- Securing your AWS credentials and account access
- Reviewing and hardening any code before deploying to production
- Following AWS security best practices for services like Amazon Bedrock and SageMaker

The example code in this repository is intended for learning purposes and may not include production-grade security controls such as input validation, least-privilege IAM policies, or encryption at rest.
